# PDFWix

A simple, branded GitHub Pages site for **PDFWix** — a collection of free, browser-based PDF tools. Every tool processes files entirely in your browser, so your documents are never uploaded to a server.

**Live site:** https://www.pdfwix.com/

**Live Github:** https://pdfwix.github.io/pdfwix/

## Tools featured on this page

- [Merge PDF](https://www.pdfwix.com/merge-pdf) — combine multiple PDFs into one file
- [Split PDF](https://www.pdfwix.com/split-pdf) — separate a PDF into pages or ranges
- [Remove Pages](https://www.pdfwix.com/remove-pages) — delete unwanted pages from a PDF
- [JPG to PDF](https://www.pdfwix.com/jpg-to-pdf) — turn images into a single PDF
- [PDF to JPG](https://www.pdfwix.com/pdf-to-jpg) — convert PDF pages into images
- [HEIC to JPG](https://www.pdfwix.com/heic-to-jpg) — convert iPhone/iPad photos to JPG
- [Sign PDF](https://www.pdfwix.com/sign-pdf) — add your signature to a document
- [Protect PDF](https://www.pdfwix.com/protect-pdf) — password-protect a PDF

## Why PDFWix

- Files are processed in the browser and never uploaded
- Free to use — no trials or premium gate
- No signup and no login wall
- No watermark on output
- Nothing to install
- Works on Windows, Mac, iPhone and Android

## Deploy with GitHub Pages

1. Push these files to the root of a public repository (for example, `pdfwix`).
2. Open **Settings → Pages**.
3. Under **Build and deployment**, choose **Deploy from a branch**.
4. Set the branch to `main` and the folder to `/ (root)`, then **Save**.
5. After a minute, the site is live at `https://<your-username>.github.io/<repo-name>/`.

## Notes

- Update the `canonical` and Open Graph URLs in `index.html` to match your actual GitHub Pages address.
- Only working tools are listed. Coming-soon tools are intentionally left off so visitors never click through to something unfinished.
